## Common Tasks

### Creating Your First Study Plan

1. Read [Getting Started](getting-started.md)
2. Follow steps to create a program
3. Add modules and lessons
4. Check [Dashboard](dashboard.md) to start tracking
5. See [Tips & Tricks](tips-and-tricks.md) for best practices

### Importing a Syllabus

1. Review [YAML Import Guide](yaml-import.md)
2. Prepare YAML file from course materials
3. Use Settings → Import Program
4. Verify structure on Dashboard

### Daily Study Workflow

1. Check [Dashboard](dashboard.md) for today's work
2. Open lessons and [log sessions](lesson-detail.md)
3. Mark complete when done
4. Review spaced-repetition items
5. See [Tips & Tricks](tips-and-tricks.md) for optimization

### Tracking Progress

1. Visit [Reports](reports.md) page
2. Review completion % and time spent
3. Adjust future planning based on insights
4. Use exports to analyze further

### Preparing for Assessments

1. Create assessment in [Assessments & Reviews](assessments-and-reviews.md)
2. Link to relevant lessons
3. Log prep sessions
4. Use spaced repetition to prepare
5. Track results

## Key Concepts

**Programs**: Top-level containers (Year, Term, Block, Custom)
**Modules**: Collections of related lessons
**Lessons**: Individual learning units with due dates
**Assessments**: Evaluations (tests, projects, papers)
**Reviews**: Spaced-repetition study sessions
**Sessions**: Time spent studying (tracked manually or via timer)
**Resources**: Reference materials and links
**Areas**: Categories for organizing programs/resources
**Status**: Current state (Not Started, Active, Blocked, Completed)
**Block**: Focused study period within a term

## Quick Tips

- Check Dashboard daily for your daily work queue
- Log study sessions immediately after studying
- Mark lessons complete when truly done (triggers reviews)
- Don't skip reviews—they're crucial for retention
- Use Calendar to validate your workload early
- Export Reports monthly to analyze progress
- Archive completed programs to reduce clutter
