# Reports & Analytics

The **Reports** page provides analytics about your study progress, completion rates, and time invested.

## Accessing Reports

1. Click **Reports** in the sidebar
2. View various statistics and summaries
3. Use filters to focus on specific programs

## Report Sections

### Overall Statistics

**Program Filter:**

- Select a program to see stats for that program only
- Leave blank to see overall stats
- Useful for comparing programs

### Lesson Progress

Shows the status breakdown of all lessons:

**Displays:**

- **Total Lessons**: Number of lessons in selection
- **Completed**: Number marked complete
- **Active**: Number currently in progress
- **Not Started**: Number not yet started
- **Blocked**: Number currently blocked
- **Completion %**: Percentage of lessons completed

**Use it to:**

- See overall completion status
- Understand progress through program
- Identify when you're getting behind
- Validate status tracking accuracy

### Session Statistics

Shows time invested in studying:

**Displays:**

- **Total Study Hours**: All sessions combined
- **Total Sessions**: Number of study sessions logged
- **Average Session Length**: Typical duration
- **Sessions by Program**: Breakdown by program
- **Sessions by Lesson**: Time per lesson

**Use it to:**

- Track time investment
- See which areas take more time
- Estimate how long lessons take
- Compare time vs. learning outcomes

**Note**: Requires logging sessions. More data = more insights.

### Review Statistics

Shows spaced-repetition progress:

**Displays:**

- **Total Reviews**: All reviews created
- **Due Now**: Reviews ready to do today
- **In Progress**: Reviews being worked on
- **Completed**: Reviews finished
- **Completion %**: What percentage are finished

**Use it to:**

- Monitor review schedule
- See if keeping up with reviews
- Identify overdue reviews
- Track retention practice

### Assessment Statistics

Shows evaluation progress:

**Displays:**

- **Total Assessments**: All assessments created
- **Not Started**: Haven't begun prep
- **In Progress**: Currently working on
- **Submitted**: Turned in
- **Graded**: Received results
- **Average Time to Completion**: Days from creation to submission

**Use it to:**

- Track assessment workload
- See preparation progress
- Identify late submissions
- Estimate assessment timelines

## Using Filters

### By Program

Select a program from the dropdown to filter:

- Only that program's lessons
- That program's sessions
- That program's reviews
- That program's assessments

**Use case:**

- Compare progress across different programs
- See if one program takes more time
- Validate schedule fairness

### By Time Period

Some reports show data within time ranges:

- **This Week**: Last 7 days
- **This Month**: Last 30 days
- **All Time**: All data
- **Custom Range**: Specific dates

**Use case:**

- See weekly progress
- Monthly review of work
- Compare semesters

## Interpreting Reports

### High Completion %

**Good sign if:**

- Finishing lessons on time
- Not rushing at end of program
- Sustainable pace

**Concerning if:**

- Marking complete too quickly
- Quality suffering for speed
- Need to check lesson status accuracy

### Low Session Time

**Could mean:**

- Efficient learner
- Not logging sessions accurately
- Lessons are easier than expected
- Need more practice

**Consider:**

- Are you logging all study time?
- Are lessons well-constructed?
- Do you need more practice?

### High Review Count Due

**Indicates:**

- Good learning completion
- Review schedule building up
- Need to allocate time for reviews
- System is working—reviews needed for retention

**Action:** Schedule regular review time.

### Blocked Count

**If high:**

- Dependencies between lessons
- Prerequisites not completed
- Need to unblock those items

**If zero:**

- No dependency structure
- Everything is independent
- Less planning constraint

**If low:**

- Good dependency planning
- Few blockers to work around

## Data Accuracy

Reports are only as good as your data:

### To Improve Report Quality

1. **Log sessions**: Every study session matters
2. **Update status**: Mark lessons complete when done
3. **Create assessments**: For all graded work
4. **Complete reviews**: Do scheduled reviews
5. **Use descriptions**: Add notes about work

### Common Data Gaps

- Missing sessions (forgot to log)
- Incorrect status (lesson still open after done)
- Missing assessments (didn't create entries)
- Skipped reviews (incomplete retention practice)

## Tips for Using Reports

### Regular Check-ins

- Weekly: Review this week's sessions and completion
- Monthly: Review overall progress and time spent
- End of program: Full report of program completion

### Identifying Patterns

Look for:

- Which topics take longest (complex)
- Which lessons you speed through (easier)
- Review effectiveness (retention vs. time)
- Assessment preparation time
- Optimal study session length for you

### Adjusting Your Approach

If reports show:

- **Too little time spent**: Add more practice
- **Too much time spent**: May need better resources
- **Low completion**: Check if schedule is realistic
- **High reviews due**: Need to allocate review time
- **Assessments behind**: Start prep earlier

### Setting Goals

Use reports to set realistic goals:

- "Complete 50% of lessons by midterm"
- "Average 20 hours per week"
- "Complete 5 reviews per week"
- "Submit assessments 3 days early"
